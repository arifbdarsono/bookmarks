Git Basics

Git is a distributed version control system that tracks changes in your code and enables collaboration with other developers. It stores the complete history of your project and allows you to work on different features simultaneously through branching.

Repository

A Git repository (repo) is a directory that contains your project files along with Git's tracking information stored in a hidden .git folder.

Commit

A commit is a snapshot of your project at a specific point in time. Each commit has a unique ID and contains a message describing the changes.

Initial Setup

Before using Git, configure your identity and preferences:

# Set your name and email
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# Set default branch name
git config --global init.defaultBranch main

# View all configurations
git config --list

Essential Commands

Repository Initialization

# Initialize new repository
git init

# Clone existing repository
git clone <url>

Basic Workflow

# Check repository status
git status

# Add files to staging
git add <file>
git add .

# Commit changes
git commit -m "message"

Viewing History

# View commit history
git log
git log --oneline
git log --graph --oneline --all

# Show changes in a commit
git show <commit-hash>

# View file differences
git diff
git diff --staged

Branching

Branches allow you to work on different features or experiments without affecting the main codebase.

# List branches
git branch
git branch -a

# Create new branch
git branch <branch-name>
git checkout -b <branch-name>

# Switch branches
git checkout <branch-name>
git switch <branch-name>

# Merge branches
git checkout main
git merge <branch-name>

# Delete branch
git branch -d <branch-name>

Remote Repositories

Work with remote repositories like GitHub, GitLab, or Bitbucket:

# Add remote repository
git remote add origin <url>

# View remotes
git remote -v

# Push changes
git push origin main
git push -u origin <branch-name>

# Pull changes
git pull origin main
git fetch origin

# Update from remote
git pull
git fetch && git merge

Common Workflows

Feature Branch Workflow

Step 1: Create a new branch for your feature
git checkout -b feature/new-feature
Step 2: Make changes and commit them
git add .
git commit -m "Add new feature functionality"
Step 3: Push branch to remote
git push -u origin feature/new-feature
Step 4: Create pull request and merge
git checkout main
git pull origin main
git merge feature/new-feature

Best Practices

Commit Messages
  • Use present tense ("Add feature" not "Added feature")
  • Keep first line under 50 characters
  • Be descriptive but concise
  • Use conventional commit format when possible
Repository Management
  • Use .gitignore for files you don't want to track
  • Commit frequently with logical chunks
  • Use branches for new features
  • Keep main branch stable and deployable
Important Commands for Troubleshooting
# Undo last commit (keep changes)
git reset --soft HEAD~1

# Discard local changes
git checkout -- <file>
git reset --hard HEAD

# Stash changes temporarily
git stash
git stash pop

# View stash list
git stash list
